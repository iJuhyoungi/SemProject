# 🧠 [개념 정리] i.MX RT1020 베어메탈 부팅 및 메모리 아키텍처

베어메탈(Bare-metal) 환경에서 칩에 전원이 켜지고 `main()` 함수가 실행되기까지의 핵심 개념과 구성 요소들을 정리합니다.

---

## 1. 메모리 맵 (Memory Map)
- **개념:** MCU가 가지고 있는 **'주소의 지도'**입니다. CPU가 특정 주소(예: `0x60000000`)에 접근했을 때, 그곳이 코드가 저장된 Flash(ROM)인지, 아니면 변수를 읽고 쓸 수 있는 RAM(TCM, OCRAM)인지 하드웨어적으로 정해져 있는 구역을 의미합니다.
- **역할:** `0x60000000`에는 부트 데이터를, `0x20000000`에는 초고속 RAM인 DTCM을 쓰겠다는 식으로 링커 스크립트에 설계도를 그릴 수 있는 바탕이 됩니다.

## 2. 링커 스크립트 (LD 스크립트, `.ld`)
- **개념:** 컴파일러가 만들어낸 코드와 변수 조각들을 메모리 맵의 **어느 주소에 배치할지 링커(Linker)에게 지시하는 명세서**입니다.
- **역할:**
  1. C 코드(`.text`)는 전원이 꺼져도 날아가지 않는 Flash 영역에 배치합니다.
  2. 초기값이 있는 전역 변수(`.data`)는 Flash에 저장해두고, 부팅 시 RAM으로 복사되도록 **LMA**(로드 주소)와 **VMA**(실행 주소)를 나누어 설정합니다.

## 3. IVT (Image Vector Table) & FCB (Flash Configuration Block)
일반적인 MCU와 구별되는 **NXP i.MX RT 시리즈 특유의 부팅 관문**입니다.

- **FCB (위치: `0x60000000`):** 칩에 전원이 켜지면 내부의 작은 공장 초기화 프로그램(Boot ROM)이 가장 먼저 실행됩니다. Boot ROM이 외부 Flash 메모리를 읽기 위해 알아야 하는 핀 설정, 읽기 속도, 명령어(`0xEB`) 등을 담은 **512바이트 크기의 설정표**입니다.
- **IVT (위치: `0x60001000`):** Boot ROM이 FCB를 읽고 Flash와 통신에 성공한 후, "진짜 펌웨어가 어디서부터 시작되는지" 찾기 위한 **이정표**입니다. 부팅 데이터의 크기, 위치, 그리고 스타트업 코드의 시작 주소를 Boot ROM에게 알려줍니다.

## 4. 스타트업 코드 (Startup Code)
- **개념:** Boot ROM이 IVT를 읽고 제어권을 넘겨주었을 때, `main()` 함수가 무사히 실행될 수 있도록 **칩을 준비시키는 최초의 코드**입니다.
- **역할:**
  1. 링커 스크립트가 지정한 주소를 참조하여, Flash에 잠들어 있던 `.data`(초기값이 있는 변수)를 RAM으로 복사합니다.
  2. `.bss`(초기값이 0인 변수) 영역의 RAM 공간을 0으로 싹 청소합니다.
  3. 이 청소가 끝나야 비로소 우리가 아는 C 언어의 `main()` 함수로 점프(호출)할 수 있습니다.

## 5. VTOR (Vector Table Offset Register)
- **개념:** ARM Cortex-M 코어 내부에 있는 핵심 레지스터로, "인터럽트(예외 상황, 타이머, 통신 수신 등)가 발생했을 때, **몇 번 주소로 달려가서 처리 코드를 찾아야 하는지**" 그 인터럽트 테이블의 시작 주소를 보관하는 곳입니다.
- **역할:** 기본적으로 CPU는 `0x00000000`에서 이 테이블을 찾으려 합니다. 하지만 우리의 코드는 외부 Flash(`0x60002000` 대역)에 있으므로, 스타트업 코드에서 `SCB_VTOR`에 우리 테이블의 절대 주소를 덮어씌워 주어 인터럽트 발생 시 칩이 미아(HardFault)가 되지 않도록 방지합니다.

## 6. FPU (Floating Point Unit)
- **개념:** 소수점 연산(`float`, `double`)을 소프트웨어로 계산하지 않고, 하드웨어 칩이 초고속으로 직접 계산하게 해주는 **하드웨어 가속기**입니다. (Cortex-M7 내장)
- **역할:** 컴파일러(`-mfpu=fpv5-d16`)는 이미 FPU 하드웨어가 있다고 가정하고 소수점 명령어를 생성합니다. 하지만 전원이 켜진 직후 FPU는 'OFF' 상태이므로, 스타트업 코드 초반에 레지스터(`SCB_CPACR`)를 건드려 FPU 하드웨어를 'ON' 상태로 켜주어야 합니다.

---

## 💡 🚀 전체 부팅 시퀀스 요약 (Boot Sequence)

1. **전원 ON** ➡️ 칩 내부의 Boot ROM이 깨어남.
2. Boot ROM이 메모리 맵 `0x60000000`의 **FCB**를 읽고 외부 Flash를 인식.
3. 이어서 `0x60001000`의 **IVT**를 읽고 진입점 파악.
4. **스타트업 코드**(`Reset_Handler`)로 제어권이 넘어옴.
5. 스타트업 코드가 하드웨어 **FPU**를 켜고, 인터럽트 주소(**VTOR**)를 올바른 위치로 바로잡음.
6. **LD 스크립트**의 지시대로 초기화 변수들을 Flash에서 RAM으로 복사.
7. 모든 하드웨어 및 메모리 준비 완료 후, 드디어 **`main()` 함수 실행!**
