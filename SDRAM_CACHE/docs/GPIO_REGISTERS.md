1. CCM_CCGR1 (0x400FC06C) : 클럭 공급 (심장 박동 켜기)

    공식 명칭: Clock Control Module - Clock Gating Register 1

    하는 일: 칩 내부의 수많은 모듈(GPIO, UART, USB 등) 중에서 특정 모듈에만 선택적으로 전원(클럭)을 공급하는 역할을 합니다.

    비유: 아무리 좋은 기계가 있어도 공장의 두꺼비집(메인 스위치)을 안 올리면 전기가 안 들어와서 안 켜지겠죠? i.MX RT 칩은 배터리를 아끼려고 처음엔 모든 공장(모듈)의 전원을 다 꺼놓고 시작합니다.

    코드의 의미: 주형님이 겪으셨던 Cannot access memory 에러의 원인이 바로 이겁니다. 이 코드가 실행되어 GPIO1 모듈에 전기를 넣어주기 전까지는 칩이 메모리 접근을 튕겨내 버리는 것이죠.

2. IOMUXC_SW_MUX_CTL... (0x401F80D0) : 핀의 역할 선택 (멀티플렉싱)

    공식 명칭: IOMUX Controller - Software Mux Control Register

    하는 일: 물리적인 칩의 다리(Pin) 하나는 보통 5~8개의 다재다능한 능력을 갖추고 있습니다. (예: 어떤 때는 UART 통신 핀, 어떤 때는 I2C 핀, 어떤 때는 단순 GPIO 핀). 이 레지스터는 **"이 핀을 지금 어떤 역할로 쓸 것인가?"**를 결정합니다.

    캡처 화면 분석: 주형님이 올려주신 화면 하단을 보면 **MUX_MODE**라는 필드가 있죠? 여기에 0부터 7까지의 숫자(ALT 모드)를 넣을 수 있습니다.

    코드의 의미: 주형님이 = 0x05; 라고 넣으신 것은, 데이터시트 상의 **ALT5 모드(단순 입출력인 GPIO 모드)**로 이 핀을 쓰겠다고 칩에게 명확히 선언한 것입니다. (맥가이버칼에서 수많은 도구 중 '십자드라이버'를 쏙 뽑아든 것과 같습니다.)

3. GPIO1_GDIR (0x401B8004) : 입출력 방향 설정 (문 열기)

    공식 명칭: GPIO Direction Register

    하는 일: 핀을 GPIO로 쓰겠다고 정했다면, 이제 이 핀이 신호를 받아들일지(Input - 예: 버튼 누름 감지), 아니면 신호를 내보낼지(Output - 예: LED 켜기) 방향을 정해야 합니다.

    코드의 의미: 주형님이 |= (1 << 5)를 하신 건 5번 핀의 방향 비트를 1로 세팅한 것입니다. 데이터시트 규격상 1은 출력(Output), 0은 입력(Input)을 의미합니다. "이 핀으로는 전류를 내보내겠다!"라고 길을 뚫어준 것이죠.

4. GPIO1_DR (0x401B8000) : 실제 데이터 조작 (스위치 켜고 끄기)

    공식 명칭: GPIO Data Register

    하는 일: 앞선 3단계를 거쳐 완벽하게 세팅된 핀에 실제로 3.3V(HIGH)를 쏠 것인지, 0V(LOW)를 쏠 것인지 결정하는 최종 스위치입니다.

    코드의 의미: 주형님이 GDB에서 뚫어져라 관찰하셨던 바로 그 레지스터입니다! 이 레지스터의 5번 비트를 1로 만들면 보드의 LED 핀으로 3.3V 전류가 흘러나가 불이 켜지고, 0으로 만들면 전압이 0V로 떨어져 불이 꺼집니다.
